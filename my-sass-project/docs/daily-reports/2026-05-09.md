## 2026/05/09 Day10 Vue3 深度学习日报

+ 学习时长：3 小时

+ 学习内容：
  - Vue3 ref/reactive 源码级原理：RefImpl 类的 getter/setter 依赖收集与派发更新机制，手写 15 行 mini ref 实现
  - Vue3 computed 底层原理：computed = ref + dirty 缓存标记，懒执行机制，结合 useForm.ts 中 isDirty 逐行推演
  - Vue3 watch 底层原理：watch = 手动声明依赖的 effect，实战添加了草稿自动保存的 watch（deep: true）
  - Vue3 递归组件通信：深度剖析 BuilderNode.vue，递归终止条件、parentList 职责、事件冒泡透传、key 用 id 非 index、.stop 修饰符
  - TypeScript 泛型实战：useForm<T>、useTable<T, P extends BasePageParams>、FormComponent 递归类型、Partial<SystemUser>

+ 心得与收获：
  今天最大的突破是从会用 API 进入了理解为什么这样设计的层次。ref 通过 get value()/set value() 劫持读写实现依赖收集和派发更新，这是唯一能代理原始值的方式，因为 Proxy 只能代理对象。computed 的 dirty 标记机制让 isDirty 不会每次都执行 JSON.stringify——懒执行只在被读取且 dirty=true 时才重新计算。递归组件的学习含金量最高——BuilderNode 的终止条件不是靠 key，而是靠 v-if/v-else-if/v-else 三分支，parentList 不是数据副本而是引用，子节点通过它知道拖入目标数组。今天学习方式也从单向阅读改为对话式，先让自己回答再纠正，暴露盲区再填补效率高很多。
