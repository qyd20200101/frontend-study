## 一、我真正掌握的内容（🟢）

- ref/reactive 核心是一致的，ref 只是多了对基本类型包装和 .value 调用
- 编译过程有 patchFlag 动态标记、静态提升、事件缓存
- diff 双端查找和最长递增子序列（今天再看才记起来）
  - 使用双端diff:保证DOM move操作次数最少
- 运行时阶段是 diff 比较

## 二、我模棱两可的内容（🟡）

- Vue2 vs Vue3 响应式机制区别：Vue2 通过 defineProperty 实现响应式，无法新增/修改数据只能通过 $set 和 $delete，数组只能重写七个方法；Vue3 用 Proxy 代理
  - reactive多了一层`Map<target,Map<key,Set<effect>>>`三层嵌套（targetMap）,因为对象有多个key,ref因为只有一个值,所以一个def就够了
- computed 的底层原理："带返回值的 effect"，不知道对不对
  - 关键：实质是一个带缓存的effect,返回ref(.value)访问,读.va;ue按需算（lazy_dirty标记）,dirty =true，不立即重算
- watch
  - 一个手动控制的effect，返回stop函数（用来停止监听）,依赖变旧立即或（flush）跑回调
  - scheduler把cn推入scheduler队列
- scheduler 和 nextTick：看到下午题目才想起来
  - nextTick(cb)= Promise.resolve().then(cb),和scheduler共享一个微任务队列，但effect已经先入队，所以cb拿到的是更新后的DOM
  - effect只入队一次->微任务只跑一次
- v-for 不能用 index 做 key：看到题目才回顾起来
  - 列表头部插入新元素，其余元素index后移，导致vue认为每个节点改变，走patch，重新渲染编译
  - 使用业务id,让diff能识别还是同一个节点,走move
- effect / dep / track / trigger 在底层代码如何实现可以说没记住
  - 读：.value->getter->track->dep.add(activeEffect)
  - 写:.value->setter->trigger->.dep.forEach(eff=>scheduler(eff))

## 三、我没记住的内容（🔴）

- 编译、挂载、渲染、更新四个阶段的具体内容
  - 模板字符串.vue->render函数->真实DOM+effect关联(render函数本身就是一个effect)
  - 编译是构建时的事情(产出render+标记)
  - 编译：compile,构建时vue-loader,/vite-plugin-vue完成
  - 挂载：(mount，首次执行render->拿到VNode->patch(null,vnode))
  - 渲染：(render,每次effect重跑,产出新vnode)
  - 更新：(patch,新旧vnode diff->局部DOM操作)
  - 数据变化:trigger->queueJob->微任务里跑effect->回到渲染
- 生命周期钩子：组件和页面的，执行前后，挂载渲染 DOM 更新——"很混乱"
- 组件钩子顺序：父子嵌套时，记住父开门让子进来,子先穿衣，父后穿衣
  - setup->onBeforeMount->onMounted->onBeforeUpdate->onUpdated->onBeforeUnmount->onBeforeUmount->onUnmounted
  - 指令钩子函数：created->beforeMount->mounted->beforeUpdated->updated->beforeUnmount->unmounted
  - 指令mounted在元素插入DOM后立刻触发，组件onMounted在所有子元素（含指令）挂载完成后触发,指令mounted->组件onMounted（子先父后，指令算更靠近DOM的子）
- watch 没有返回值并且是手动的 effect，和 track/trigger 数据更新的联系
- v-if / v-for 等价于什么
  - v-if编译为三元表达式
  - v-for编译为map函数
- v-model 等价于什么，和 Vue2 v-model 的区别
  - Vue3 :modeValue+@update:modelValue支持多个v-model:title
  - Vue2 :value+@input,改名要用model和.async
- TS：类型泛型、约束、工具类型"一窍不通"，场景实现"一塌糊涂"（如传参取固定值使用 TS）
  - 泛型：函数/类的类型参数
  - 约束extends:T必须是某种形状
  - 工具类型
    - Partial<T>全部可选
    - Required<T>全部必填
    - Pick<T,K>挑选K这几个key
    - Omit<T,K>排除K这几个key
    - Record<K,V>等价{[k in k]:V}
  - 接口传参场景
    - 字面量+泛型约束
    - type: Theme = 'light'|'dark',function setTheme(t:Theme){...}只能传这两个值
    - 进阶：从obj里只允许传它的某个key
      - function get<T,K extends keyof T>(obj； T，key:K):T[K]{...}
    - interface vs type
      - interface可被extends/implement，可合并声明
      - type 可联合/教程，不可合并
