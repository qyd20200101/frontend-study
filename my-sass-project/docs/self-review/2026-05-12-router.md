# 2026-05-12 Vue Router 学习 — 自我总结

---

### 🟢1 — hash vs history 模式区别

**核心**：hash 模式 URL 带 #，history 模式不带 # 需要服务器配置

| 模式 | URL | 服务器需要 |
|---|---|---|
| createWebHashHistory | /#/login | 不需要 |
| createWebHistory | /login | 需要重定向到 index.html |

**场景**：你项目用 createWebHistory

---

### 🟢2 — 路由匹配优先级（404 必须在最后）

**核心**：先注册先匹配，404 最后加，防止拦截动态路由

**场景**：你 guard.ts 中 accessRoutes.forEach(addRoute) 之后再 addRoute 404

---

### 🟢3 — next({ path }) vs next(string)

**核心**：效果相同，对象形式支持额外配置（replace/query）

| 形式 | 用法 |
|---|---|
| next('/') | 简单跳转 |
| next({ path: '/', replace: true }) | 替换历史 + 跳转 |

> 易错点：误以为 replace: true 是"确保添加成功"

**场景**：F5 刷新后动态路由重定向

---

### 🟢4 — replace: true 含义

**核心**：用新导航替换当前历史记录条目，不追加

```ts
// 不加 replace → 历史: [..., /dash, /dash]
// 加 replace   → 历史: [..., /dash]
```

**场景**：F5 刷新重拉用户信息后 next({ ...to, replace: true })

---

**核心**：全局守卫只是 7 步中的 2 步（beforeEach/afterEach），还有路由独享和组件内守卫

| 5 | 解析异步组件 | 自动 |
| 6 | beforeRouteEnter | 组件内（进入前） |
| 7 | afterEach | 全局 |
> 易错点：以为只有 beforeEach/afterEach，漏掉组件内守卫和路由独享守卫
