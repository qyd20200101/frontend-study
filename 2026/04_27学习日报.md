## 2026/04/27 Day26学习日报

+ 学习内容：
  + proxy
    + 在遍历对象时只会代理第一层，当访问嵌套属性值时才会为具体的动态代理
    + Proxy原生支持新增/删除属性,Vue2不能监听delete，新加属性要用$set
    + Proxy可以直接监听数组具体值和长度,Vue2必须重写数组方法获取
    + Proxy支持Map/Set,Vue2无法拦截集合
  + ref
    + 底层：
      + 接受原始类型，通过js对象使用get value(),set value()属性服访问器拦截
      + ref接受对象时，内部调用reactive
      + js，原始类型按值传递,let x = 10;赋值，无法追踪变化
      + 为了实现响应式：把数据包装在对象里面，通过引用传递触发依赖收集
  + 依赖存储
    + WeakMap作为根节点，，防止内存泄露，Map属性层，为对象具体属性名，Set为存储所有依赖属性的effect:副作用函数，目的是去重，一个属性在一个组件多次访问只用渲染一个effect
  + activeEffect 是“当前正在运行的副作用函数”的指针。Vue 利用它来在“收集阶段”把组件渲染函数挂到对应的响应式字段上；在“派发阶段”，通过这个关联找到需要更新的渲染函数。
  + lazy缓存
    + Computed 内部使用 lazy Effect + dirty 标记，实现了访问才计算、变化才清空缓存的高效机制
  + Vue3编译器
    + 静态提升
      + Vue3编译模板会将静态资源的创建过程提升到渲染函数之外
        + 静态节点永远只会创建一次，diff过程，静态提升的VNode会被跳过比较
      + Vu2则会把所有节点重新创建一次
    + 补丁标记
      + Vue2diff算法会把所有属性内容对比
      + Vue3在编译阶段，会把动态绑定的属性打上数字标记，算法运行到该节点时只会检查动态属性
    + Vue2采用双端diff，（旧头，旧尾，新头，新尾）四个指针向中间收缩,对于乱序排列,效率会退化  
    + Vue3采用贪心算法，找到相对位置不变的最长子序列，再把剩余的插入到其中
  + Hook
    + 两个hook里面声明相同变量会发生覆盖,不清楚来源，a,bHook相互依赖
    + const{assetList} = useAsset()通过函数返回，显式解构
    + 命名自由
    + 逻辑解耦,useXXX为独立函数
  + Pinia
    + 因为vue2基于object.defineProperty，无法感知具体修改的属性和操作者,只有通过同步函数检查遍历
    + Vue3 proxy 通过set,get拦截任何属性修改，并通过状态追踪effect副函数监听变化，pinia基于vue3的proxy响应式
    + 本质：是一个全局单例响应式模块,state底层就是reactive(),getters就是computed(),Action就是普通函数
    + pinia访问state时，不需要.value获取，内部自动使用reactive对状态对象进行包装，引用解包
    + 插件系统：利用vue3 的effectScope副作用作用域，当Store被销毁，产生的所有属性监听都会被一键清除
  + 响应式系统架构（核心）：
  + 拦截机制： 深入对比 Proxy 与 Object.defineProperty。理解了 Vue 2 必须通过**原型方法劫持（Method Patching）**改写 7 个变异方法来监听数组，而 Vue 3 原生支持数组索引与长度变化的底层逻辑。
  + 依赖追踪（Track/Trigger）： 掌握了响应式的双向流程。在 get 中执行 track（收集），在 set 中执行 trigger（派发）。
  + 存储结构： 梳理了 WeakMap -> Map -> Set 的“俄罗斯套娃”式存储模型。理解了 WeakMap 以对象为键实现弱引用，避免内存泄露的精妙设计。
调度与优化：
    ** activeEffect：** 明确了其作为全局单例指针的角色，是连接“响应式数据读取”与“副作用函数注册”的唯一桥梁。
  + Computed 智能缓存： 理解了利用 lazy effect 配合 dirty（污染标记）实现的“被动触发+缓存”机制。
编译器与算法优化：
  + 精准更新： 学习了 PatchFlag（补丁标记） 如何实现动态属性的定向更新，以及 Static Hoisting（静态提升） 如何在 5 万条数据的极端场景下减少 VNode 创建开销。
  + Diff 优化： 掌握了 最长递增子序列（LIS） 算法在乱序对比中实现 DOM 移动次数最小化的原理。
  + 工程化实践：
    + 状态管理进化： 剖析了 Pinia 移除 Mutation 的底层底气（基于 Proxy 的同步追踪），以及其作为 EffectScope 管理状态生命周期的设计。
    + 高级性能权衡： 明确了在处理大数据量时，不应盲目追求全量响应式，而应配合 shallowRef 或 Object.freeze() 进行性能减负。

  + 从“使用者”向“开发者”的思维转变： 以前只知道 ref 能变，现在明白了背后 activeEffect 是如何像“窗口登记”一样完成依赖收集的。
