+ 1. Vue 3 中使用 <KeepAlive> 结合 vue-router 实现多页签缓存时，经常会遇到缓存失效的问题，根本原因是什么？在 <script setup> 语法下，你通常如何解决组件命名的问题？
    + 考察点：Vue 虚拟 DOM 的缓存匹配机制、Vue 3 新特性与兼容性。
    + 破局思路：点出 <KeepAlive> 的 include 属性严格依赖路由的 name 与组件自身的 name 完全一致。说明 <script setup> 默认匿名的痛点，并给出你的解决方案（使用双 <script> 标签声明，或使用 Vue 3.3+ 的 defineOptions 宏）。

+ 2. 在复杂的 B 端后台（如资产管理大盘）中，如何利用 CSS 的包含块（Containing Block）原理，实现一个“只遮挡右侧内容区，而不遮挡左侧全局菜单栏”的局部弹窗？

    + 考察点：CSS 定位上下文的深度理解（这是很多前端工作两三年还会晕的盲区）。

    + 破局思路：准确说出 position: fixed（寻找视口）和 position: absolute（向上寻找第一个非 static 定位的祖先元素）的区别。强调为右侧主容器加上 position: relative 这把“锁”的关键作用。

##### 🛡️ 架构与网络安全篇
3. 如果用户遇到弱网环境，连续狂点“确认报修”按钮，前端如何从底层机制上拦截重复请求，防止产生脏数据？请对比一下传统的防抖（Debounce）与 Axios 底层拦截的区别。

    + 考察点：企业级网络封装能力、AbortController 的应用。

    + 破局思路：传统防抖只是延迟执行，治标不治本；高级做法是在 Axios 的请求拦截器中维护一个 Pending 队列（Map），利用生成的 Request Key 查重。如果发现完全相同的请求正在飞，立刻使用 AbortController 斩断前一个请求，确保只有最后一次请求到达服务器。

+ 4. 在 RESTful 架构下，针对“多条件联合分页查询”和“批量删除”，你的前后端接口（HTTP Method 与参数传递）是如何设计和规范的？

    + 考察点：HTTP 协议基础、前后端联调规范。

    +   破局思路：明确指出 GET 请求（如获取分页 page）不能携带 Body，参数必须通过 URL 拼接（在 Express/Axios 中体现为 req.query / params）；而批量删除属于破坏性操作，通常使用 DELETE 或 POST 请求，大量 ID 必须放在请求体（req.body / data）中传递。

    + 考察点：TypeScript 严格模式下的泛型约束与类型断言。

    + 破局思路：解释 TypeScript 的类型保护机制（编译器无法确定 P 是否包含某些特定属性）。给出你的实战解法：使用双重断言（Double Assertion）as unknown as P 来绕过编译检查；或者提出更优雅的面向对象解法：让泛型 P 显式继承一个 BasePageParams 接口（extends）。
