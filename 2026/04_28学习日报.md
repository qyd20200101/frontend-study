## 2026/04/28 Day27学习日报
+  每天提交日报，日报格式：
+ 学习时长： 7小时
+ 学习内容： 
 + Vue 3 核心架构深度复盘与组件高级实战
  + 响应式底层逻辑（纠正与加深）：
    + Proxy vs Object.defineProperty： 纠正了 Vue 2 无法监听数组索引和属性新增的缺陷。理解了 Vue 3 Proxy 的“懒代理”机制，即只有访问到深层属性时才按需创建响应式，大幅提升了 5 万条大数据量下的初始化速度。
    + 依赖存储结构： 明确了 WeakMap(target) -> Map(key) -> Set(effects) 的三层存储模型。
    + computed 缓存（脏检查）： computed 本质是惰性 effect。利用 dirty 开关实现缓存：依赖变了仅将 dirty 设为 true，不立即计算；只有下次读取时才根据 dirty 重新计算。
  + API 深度对比：
    + ref vs reactive： ref 负责基本类型（通过 value 属性拦截），对象则交给 reactive（通过 Proxy 拦截）。.value 的设计是为了在 JS 传递中保持“响应式引用”，避免结构丢失。
    + watch vs watchEffect： watch 是精确监听，支持 immediate（立即执行）、deep（深度监听）、获取 oldValue；watchEffect 是自动追踪，首次立即执行，逻辑更简洁但无法直接对比旧值。
  + 生命周期与 DOM 操作：
    + setup 顶层： 代替了 beforeCreate/created。此时 DOM 还没渲染，不能操作元素。
    + onMounted： DOM 挂载完成，是操作 ECharts、获取 DOM 宽高的黄金时机。
    + onUpdated： 避坑点——避免在其中直接修改响应式数据，否则会触发死循环（数据变 -> 触发渲染 -> 触发 updated -> 又改数据 -> 又触发渲染）。
  + 组件通信与高级指令：
    + v-model 升级： Vue 3 将 .sync 与 v-model 统一。默认绑定 modelValue，自定义绑定 v-model:propName，支持多双向绑定。其本质仍是 props + update:xxx 事件。
    + provide/inject： 纠正术语——由 provide 发射，inject（而非 reject）接收。解决了“属性穿透（Prop Drilling）”问题。
    + Teleport： 解决了弹窗、Toast 被父级 overflow:hidden 或 z-index 限制的 UI 痛点。
    + KeepAlive： 利用 LRU 算法缓存组件实例，通过 onActivated/onDeactivated 管理休眠与唤醒。
+ 心得与收获：
    + “异步”是 Vue 性能的灵魂： 以前觉得 nextTick 麻烦，现在明白没有异步批处理，频繁刷 DOM 网页会瞬间崩溃。
    + 架构师视角的权衡： 明白什么时候该用 props（追求代码清晰可追溯），什么时候该用 provide（追求解耦）。
