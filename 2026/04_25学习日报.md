    + **全链路 Debug 意识**：今天在处理 `editor_zhang` 登录 401 问题时，通过后端日志精准定位到了 Token 生成与解析中“下划线分割”的逻辑冲突。学会了通过 $O(1)$ 的日志追踪代替盲目的代码猜测。

+  **遇到的问题 (可选)**： 
    + **问题 1**：`/api/user/info` 返回 401 导致登录后白屏。
        - **产生原因**：Token 解析逻辑使用了 `split('_')[1]`，无法处理用户名中自带下划线（如 `editor_zhang`）的情况。
        - **解决过程**：将解析逻辑重构为 `slice(1, -1).join('_')`，实现了对复杂用户名的完美兼容。
    + **问题 2**：工具栏 UI 在标签增多时发生形变。
        - **产生原因**：`padding` 设置过大（80px 笔误）且内部元素缺乏缩放保护。
        - **解决过程**：修正内边距，并引入 `filter-tags-area` 容器配合 `overflow-x: auto`，确保了在多维过滤条件下布局的稳定性。
    + **问题 3**：TS 报错“元素隐式具有 any 类型”。
        - **解决过程**：在接口中添加了字符串索引签名 `[key: string]: any`，并在排序逻辑中使用 `key as keyof Project` 进行类型锁定。
