### 2. 浏览器 / 计算机网络：缓存专项复习
今天复习了浏览器缓存机制，重点是强缓存、协商缓存和常见缓存字段。

- 强缓存：`Cache-Control / Expires`，命中后直接读本地缓存
- 协商缓存：`ETag / Last-Modified`，服务器判断资源是否变化
- `304 Not Modified`：表示资源未变化，继续使用本地缓存
- `F5`：会重新验证缓存，可能命中 304
- `Ctrl + F5`：强制重新请求，绕过缓存
- `index.html` 不适合长缓存，静态资源更适合强缓存

### 1. 链表题高频易错点
- `dummy` 的作用要明确：统一处理头结点
- 快慢指针类题目要记住循环条件：`fast !== null && fast.next !== null`
- 相交链表判断的是节点引用，不是节点值
- 删除倒数第 N 个节点时，`fast` 要先走 `n + 1` 步

### 2. 浏览器缓存容易混淆的点
- `Cache-Control` 是强缓存，`ETag / Last-Modified` 是协商缓存
- `no-cache` 不是不缓存，而是每次都要和服务器确认
- `no-store` 才是完全不缓存
- `304` 表示资源没变，不返回正文

## 学习心得

今天的学习重点是把“理解”进一步变成“模板化表达”。

在链表专题中，我对 5 道经典题的认知更加稳定了：
- 环形链表靠快慢指针判断是否相遇
- 合并有序链表靠 dummy 和双指针拼接
- 中间结点靠快慢指针定位
- 相交链表靠 A+B / B+A 补平长度差
- 删除倒数第 N 个结点靠 dummy 和找前驱

这些题虽然形式不同，但本质上都是在用固定的指针模型解决问题。今天最大的收获是：我开始能把这些题归纳成可复用的套路，而不是只记单题答案。

在浏览器缓存部分，我也把之前比较零散的知识串起来了。现在我对一次请求的完整缓存流程更清晰：
1. 第一次请求返回 200
2. 浏览器保存资源和缓存头
3. 第二次请求先看强缓存
4. 强缓存没命中再走协商缓存
5. 没变化返回 304，有变化返回 200

这样理解后，缓存不再只是几个头字段，而是“浏览器如何决定是否重新下载资源”的完整流程。
