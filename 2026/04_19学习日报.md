+ 学习内容： 
    + 作用域与闭包
        + 作用域链：执行代码，查找变量的路径
        + 闭包：函数在其外部作用域执行，保存其定义的作用域和变量
            + 例子：return function()   
        + 变量提升：var,function声明会提升，let/const存在暂时性死区
    + 浏览器渲染流程
        + 渲染路径：html到DOM,css——CSSOM——Render tree——Layout(回流)——paint(重绘)
        + 回流：元素尺寸，布局发生变化,width,height,fontSize宽高字体等发生改变
        + 重绘：元素外观如颜色背景颜色发生改变，但是布局不变的
        + 虚拟列表渲染5万条数据不会卡死：
            + 直接渲染：浏览器计算5万DOM的layout，线程被撑爆
            + 虚拟：transform是在合成层处理，不触发回流和重绘，直接又GPU加速
    + 计算机网络
        + TCP: 
            + 三次握手：客户端——>服务端——>客户端,关键标志位：“SYN（请求连接）-> SYN+ACK（确认并响应）-> ACK（最终确认）”。
            + 四次挥手：断开链接，因为服务端可能有数据没传完
            + TCP重传机制，拥塞控制，UDP快（直播，游戏用）
    + 路由守卫
        + router.beforeEach在跳转页面钱拦截，检查身份,利用 router.addRoute() 实现路由的动态注入
        + 流程：进度条开始，白名单检查，Token检查
    + 动态路由
        + 防止用户在地址栏强行输入跳转
        + 流程：防止越权访问。流程为：获取用户角色 -> 递归过滤动态路由表 -> `router.addRoute` 注入 -> 确保 404 路由最后挂载
    + 无痛刷新与Token持久化
        + 利用 `isRefreshing` 锁和 **Promise 闭包队列**。将并发请求封装在匿名函数中，通过返回未 resolve 的 Promise 使其进入 pending 状态。Token 刷新成功后遍历队列 resolve，实现请求重发
