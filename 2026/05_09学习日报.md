## 2026/05/09 Day10 Vue3 深度学习日报
+  每天提交日报，日报格式：
+ 学习时长：8小时
+ 学习内容： 今天主要完成了半月总复盘，并系统复习了 Vue3 的核心知识体系。Vue3 部分按照“响应式系统 → 页面编译渲染 → 数据传递 → v-model 语法糖 → 事件循环与 DOM 更新 → KeepAlive 页面状态保持”这条主线进行了梳理，重点明确了 reactive 与 ref 的区别、computed/watch/watchEffect 的作用和差异、nextTick 与 DOM 更新的关系、props/emit / provide-inject / Pinia 的适用场景，以及 v-model 在 Vue3 中支持多个绑定、替代 .sync 的语法变化。同时复习了 Vue3 的编译优化，包括 template 到 render 的编译链路、静态提升、patchFlag 的作用，以及 Vue3 相比 Vue2 在响应式、编译和更新调度上的改进。
  - Vue3 ref/reactive 源码级原理：RefImpl 类的 getter/setter 依赖收集与派发更新机制，手写 15 行 mini ref 实现
  - Vue3 computed 底层原理：computed = ref + dirty 缓存标记，懒执行机制，结合 useForm.ts 中 isDirty 逐行推演
  - Vue3 watch 底层原理：watch = 手动声明依赖的 effect，实战添加了草稿自动保存的 watch（deep: true）
  - Vue3 递归组件通信：深度剖析 BuilderNode.vue，递归终止条件、parentList 职责、事件冒泡透传、key 用 id 非 index、.stop 修饰符
  - TypeScript 泛型实战：useForm<T>、useTable<T, P extends BasePageParams>、FormComponent 递归类型、Partial<SystemUser>

+ 心得与收获： 今天最大的收获是对 Vue3 的理解从“零散 API 记忆”升级到了“体系化骨架理解”。以前我容易把 ref、reactive、computed、watch、watchEffect、nextTick、KeepAlive 等知识点当成孤立概念去记，导致回答时经常只能说出模糊印象，缺少上下文链路。通过今天的系统复盘，我逐渐建立起了 Vue3 的完整脉络：数据从响应式声明开始，经由 track/trigger 完成依赖收集和更新触发，模板经过编译得到 render 函数，再通过 patch/diff 更新真实 DOM，而组件间的数据流动则通过 props/emit、v-model、provide/inject 和 Pinia 在不同层级之间传递，页面状态则可以通过 KeepAlive 缓存实例来保持。这个过程让我第一次把 Vue3 作为“一整套从源码到底层渲染再到项目落地”的体系来理解，而不只是一个个 API 的堆叠。
  在 Vue3 与 Vue2 的对比上，我也更加清晰地认识到核心差异：Vue2 依赖 Object.defineProperty，需要递归遍历和重写数组方法，而 Vue3 使用 Proxy，天然支持新增删除属性和数组操作，并通过懒响应提升初始化性能；编译阶段 Vue3 通过静态提升和 patchFlag 精准更新动态节点，避免全量 diff；更新阶段则通过微任务队列批量更新 DOM，再结合 nextTick 保证更新后获取到最新 DOM。今天的学习不仅让我理解了“是什么”，也开始能说出“为什么这样设计”“在项目里怎么用”“遇到问题该选哪种 API”，这对后续写代码和面试表达都有很大帮助。
+ 遇到的问题 (可选)： 今天在 Vue3 的源码层理解上还停留在“知道有 track/trigger、activeEffect、scheduler 这些机制”，但还没有做到完全把简化实现和源码一一对应；另外对于编译阶段 patchFlag 的细节、以及实际项目中如何根据场景选择 computed/watch/watchEffect 仍需要继续通过项目和源码加深理解。
