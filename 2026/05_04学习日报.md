+ 学习内容：
  - 回顾并梳理了二叉树专题的核心题型，包括：
    - 二叉树最大深度
    - 对称二叉树
    - 二叉树层序遍历
    - 路径总和
    - 最近公共祖先
  - 重点进行了二叉树 DFS / BFS 对照训练，明确了两种遍历方式的职责差异：
    - DFS：适合路径、子树、递归汇总类题
    - BFS：适合层级、队列、按层处理类题
  - 学习并复盘了回溯专题的经典题型：
    - 全排列
    - 子集
    - 组合

  - 通过对回溯题的梳理，进一步明确了回溯的核心模板：
    - 路径 path
    - 选择列表
    - 结束条件
    - 撤销选择
  - 对比总结了全排列、子集、组合的区别：
    - 全排列需要 used 数组
    - 子集和组合使用 start 控制范围
    - 组合是在子集基础上增加固定长度限制
  - 整理了树题和回溯题的面试答题思路，形成了“题型识别 → 核心思路 → 边界条件 → 代码实现 → 复杂度”的统一表达方式

+ 心得与收获：
  今天的学习内容比日报里最初记录得更完整，除了二叉树专题外，还系统复盘了回溯专题。通过全排列、子集、组合和括号生成这几道题，我对回溯的整体模板理解更清晰了。

  回溯题的核心不再是单纯背题，而是要抓住三个关键点：路径、选择和撤销。全排列强调顺序，所以需要 used 数组；子集和组合不强调顺序，因此通过 start 控制向后选即可；组合则是在子集的基础上加了固定长度 k 的约束；括号生成则是带剪枝条件的构造问题，需要通过左右括号数量限制保证结果合法。

  二叉树部分今天也有很大收获。相比之前只“知道道理”，今天对 DFS 和 BFS 的职责划分更明确了：DFS 解决的是递归拆分和状态汇总问题，比如最大深度、路径总和、最近公共祖先；BFS 解决的是按层处理问题，比如层序遍历和按层统计深度。尤其是 BFS 里的 size 切层概念，对我理解层序遍历非常关键。

  总体来看，今天的学习让我开始从“会做单题”走向“会归纳题型”。不管是树还是回溯，我都在逐步形成自己的模板意识，这对后续面试非常重要。

+ 遇到的问题：
  - 二叉树题型的递归模板还需要继续熟练，尤其是最近公共祖先这类“返回值为节点”的题目
  - 回溯题中“什么时候收集结果、什么时候撤销选择”还需要多练几遍
