## 2026/04/29 Day24学习日报

+ **学习时长：** 7小时

+ **学习内容：** 计算机网络、浏览器原理、页面性能排查与基础算法题推进

    + **计算机网络与 HTTPS / TLS：**
        + 深入梳理了 HTTPS 的握手流程，明确了证书校验依赖的是 **CA 公钥** 对数字签名进行验证，而不是浏览器持有 CA 私钥。
        + 理解了 TLS 握手中 `Client Random`、`Server Random` 和 `Pre-Master Secret` 的作用，以及最终如何协商生成对称密钥用于后续数据传输。
        + 明确区分了“身份验证”和“数据加密”两个概念：证书用于证明服务器身份，对称密钥用于高效传输数据。

    + **TCP / UDP / HTTP / CORS：**
        + 复盘了 TCP 三次握手与四次挥手的标准流程，理解了握手的核心是**确认双方收发能力**，挥手的核心是**全双工连接的双向关闭**。
        + 明确了 `TIME_WAIT` 的作用：**保证最后 ACK 可靠送达**，以及**防止旧报文污染新连接**。
        + 对 HTTP/2 的多路复用与 HTTP/3 / QUIC 的改进有了更清晰认识，理解了 HTTP/2 在高丢包场景下可能因 TCP 队头阻塞而变慢。
        + 对 CORS 的预检请求、`Access-Control-Allow-Origin`、`Access-Control-Allow-Credentials`、`withCredentials` 等配置有了更完整的理解，明确了跨域携带 Cookie 时前后端必须配合正确配置响应头。
        + 重新梳理了 `SameSite=Lax / Strict / None` 的差异，理解了其在 CSRF 防御中的实际作用。

    + **浏览器渲染、事件循环与性能优化：**
        + 复盘了从输入 URL 到页面渲染的完整链路：`URL 解析 -> DNS 解析 -> TCP 建连 -> TLS 握手 -> 缓存判断 -> CORS -> DOM/CSSOM 构建 -> Layout -> Paint -> Composite`。
        + 理清了强缓存与协商缓存的区别，以及 `index.html`、hash 静态资源、`config.json` 在实际项目中的不同缓存策略。
        + 深入理解了浏览器的事件循环机制：同步任务、微任务、宏任务、渲染阶段的执行顺序，以及微任务不断自我创建时可能导致“饿死渲染”的问题。
        + 重点复盘了强制同步布局（Forced Synchronous Layout）与回流/重绘的关系，认识到读写交错 DOM 会触发布局抖动，影响页面性能。
        + 对 `transform: translateY` 与 `top/left` 的性能差异有了更准确的认识，理解了合成层和 GPU 加速在动画中的优势。

    + **浏览器架构与存储安全：**
        + 理解了浏览器的多进程架构：浏览器主进程、渲染进程、GPU 进程、网络进程、插件进程之间的分工与隔离。
        + 复盘了同源策略、`localStorage` / `sessionStorage` / `Cookie` 的区别，明确了 `Cookie` 的安全属性 `HttpOnly`、`Secure`、`SameSite` 的实际意义。
        + 对 `iframe` 跨域、`postMessage`、`SharedWorker`、`localStorage` 事件等跨页面通信方案有了整体认识。

    + **算法基础推进：**
        + 完成了多道基础算法题的复盘与强化：
            + **有效的括号**：掌握了栈的使用方式，理解了括号匹配的本质是“最近左括号优先匹配”。
            + **两数之和**：掌握了哈希表优化思路，理解了“空间换时间”的经典做法。
            + **反转链表**：理解了三指针 `prev / curr / next` 的作用，以及为什么必须先保存下一个节点防止断链。
            + **翻转二叉树**：理解了递归终止条件和单层逻辑，掌握了树结构的递归处理方式。
            + **最长无重复子串**：掌握了滑动窗口 + 哈希表的组合思路，理解了左边界的动态收缩逻辑。
            + **盛最多水的容器**：理解了双指针 + 贪心策略，明确了移动短板才能有机会获得更大面积。

+ **心得与收获：**
    + 今天最大的收获是把“网络协议”和“浏览器内部机制”串成了一条完整主线，不再是零散知识点，而是能按照“URL 输入 -> 建连 -> 加密 -> 缓存 -> 渲染 -> 事件循环 -> 性能排查”完整讲出来。
    + 对 HTTPS、TCP、CORS 和缓存策略的理解更系统了，尤其是证书校验、预检请求、强缓存/协商缓存这几个点，终于能从“现象”上升到“原理”和“工程配置”的层面。
    + 算法方面开始真正建立“题型模板”的感觉：栈、哈希表、滑动窗口、双指针、递归这些基础结构已经可以逐步迁移到不同题目中，说明学习进入了“会做题型”而不是“记题目”的阶段。
    + 今天也意识到自己在一些题目上容易把概念说混，比如把“缓存策略”和“TLS 握手”混在一起、把“滑动窗口”说成“贪心”，后续要继续通过标准话术和重复口述来巩固表达的准确性。

+ **遇到的问题 (可选)：**
    + **问题1：** 对 HTTPS 握手中的“随机数”概念一开始不够清楚，容易把“身份验证”和“密钥交换”混为一谈。
        + **解决：** 已明确区分：证书校验用于身份验证，Pre-Master Secret + Random 用于生成会话密钥。
    + **问题2：** 对 `SameSite=Lax`、`withCredentials`、`Allow-Credentials` 的联动配置理解不够完整。
        + **解决：** 已补齐跨域携带 Cookie 的前后端双向配置要求，并记住预检请求不携带 Cookie。
    + **问题3：** 算法题中容易把“滑动窗口”和“贪心”混淆。
        + **解决：** 已通过“最长无重复子串”和“盛最多水的容器”做了区分：前者是滑动窗口，后者是双指针 + 贪心。
