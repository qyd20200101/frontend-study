## 2026/04/22 Day19学习日报
+  每天提交日报，日报格式：
+ 学习时长： 6小时
+ 学习内容： 学习的知识点
    + 前端基础知识模拟面试 10 题实战演练：
        + `==` 和 `===`：
            + 重新梳理了宽松相等与严格相等的本质区别。
            + 明确了 `==` 会发生隐式类型转换，`===` 不会转换类型，必须类型和值都相等。
            + 重点纠正了 `null == undefined` 并不是因为它们都被转成 object，而是 JS 抽象相等比较规则里的特殊规定。
        + JavaScript 数据类型与类型判断：
            + 复习了基本数据类型：`Number`、`String`、`Boolean`、`Undefined`、`Null`、`Symbol`、`BigInt`。
            + 复习了引用类型本质都属于 `Object`，包括 `Array`、`Function`、`Date`、`RegExp` 等。
            + 强化了 `typeof`、`instanceof`、`Array.isArray()`、`Object.prototype.toString.call()` 的适用场景。
            + 纠正了 `NaN` 不是独立类型，而是 `Number` 类型中的特殊值。
        + 原型链、`instanceof` 与 `new`：
            + 重新理解了实例对象的隐式原型 `__proto__`、构造函数的 `prototype` 与原型链查找机制。
            + 明确了 `instanceof` 的底层原理是：判断构造函数的 `prototype` 是否出现在实例对象的原型链上。
            + 复习了 `new` 的四个核心步骤：创建空对象、链接原型、绑定 `this`、返回对象。
        + 作用域、作用域链、闭包：
            + 进一步区分了“变量可访问范围”“向外层词法环境查找”“函数保留外层作用域变量”这三个概念。
            + 理解了闭包的常见使用场景：异步回调、状态保存、函数封装、请求队列管理。
            + 认识到闭包如果引用大对象、DOM、定时器且不释放，可能带来内存泄漏风险。
        + `this` 指向规则：
            + 复习了普通函数、对象方法、构造函数、箭头函数的 `this` 差异。
            + 明确了对象方法遵循“谁调用指向谁”，构造函数通过 `new` 调用时 `this` 指向新实例。
            + 记住了箭头函数没有自己的 `this`，会继承外层作用域。
            + 强化了“连续 `bind` 多次，最终以第一次绑定为准”的规则。
        + 事件流与事件委托：
            + 理清了捕获阶段、目标阶段、冒泡阶段的执行顺序。
            + 明确了事件委托本质上是利用事件冒泡，将子元素事件统一交给父元素处理。
            + 总结了事件委托的优点：减少监听器数量、支持动态节点、降低内存消耗。
            + 也认识到它的局限性：不适用于部分不冒泡事件，复杂业务下父层分发逻辑容易膨胀。
        + 浏览器 Event Loop：
            + 继续巩固了“当前宏任务执行完 -> 清空微任务 -> 浏览器在合适时机渲染 -> 下一轮宏任务”的执行节奏。
            + 理解了 `Promise.then` 属于微任务、`setTimeout` 回调属于宏任务，因此前者通常先执行。
        + 从输入 URL 到页面渲染：
            + 重新串了一遍完整链路：URL 解析、缓存判断、DNS 解析、TCP 三次握手、HTTPS/TLS、HTTP 请求、DOM/CSSOM 构建、Render Tree、Layout、Paint、Composite。
            + 强化了强缓存与协商缓存的区别，以及 `ETag` / `Last-Modified` 的典型作用。
            + 纠正了“三次挥手”这一口误，建立连接应为 TCP 三次握手，断开连接才是四次挥手。
        + 5 万条数据表格综合优化题：
            + 从渲染性能、JS 单线程、虚拟列表、`KeepAlive`、事件监听与定时器清理等角度进行了系统化回答。
            + 进一步理解了虚拟列表解决的是 DOM 量过大问题，而时间切片解决的是长任务阻塞问题。
            + 强化了 `KeepAlive` 命中依赖组件 `name` 与路由 `name` 严格一致，以及缓存场景下更应该关注 `onActivated` / `onDeactivated`。
