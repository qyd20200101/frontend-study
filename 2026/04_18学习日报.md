## 2026/04/18 Day16 学习日报

+ 学习时长： 8 小时

+ 学习内容：

+ 数据类型与精度陷阱：
  + 核心原理：复习了 IEEE 754 双精度浮点数标准，理解了 0.1 + 0.2 !== 0.3 的本质是二进制舍入误差。
  + 项目应用：在资产管理系统的“预算总额”统计中，意识到必须对金额进行精度处理（先乘后除或 toFixed），防止报表数据出现长串小数。
+ 高性能数据处理（Object.freeze）：
  + 八股考点：Vue 的响应式原理是通过 Object.defineProperty (Vue2) 或 Proxy (Vue3) 递归遍历对象。
  + 实战感悟：在处理 5万条压测数据 时，利用 Object.freeze 冻结对象，告诉 Vue 引擎“不需要追踪这些数据的变化”。这一步省去了 5 万个 Proxy 代理的创建，极大地降低了内存消耗和系统初始化时间。
+  浏览器与渲染：攻克 5万条数据的卡顿
  + 时间分片（Time Slicing）：
    + 核心原理：JS 是单线程的，长耗时任务会阻塞主线程导致 UI 假死。
    + 项目应用：手写了分批次注入逻辑，利用 requestAnimationFrame（rAF）配合递归。rAF 会在浏览器每一帧刷新（约 16ms）时执行一小批任务，确保在生成 5 万条数据的过程中，Loading 动画和按钮点击依然能够得到响应。
  + 虚拟列表（Virtual List）渲染：
    + 核心原理：浏览器渲染 5 万个 DOM 节点会引发严重的重绘（Repaint）与回流（Reflow）。
    + 实战感悟：实现只渲染“可视区域”的 DOM（约 20 个）。通过监听 scrollTop 实时计算起始索引，利用 CSS 的 transform: translateY 保持滚动条的正确位置。这是解决前端大数据展示的终极方案。
+ 组件设计与通信：封装 BaseModal 与 Tree
  + Vue3 组件协议（v-model）：
    + 核心原理：深度理解了 Vue3 的单向数据流。
    + 项目应用：在 BaseModal 封装中，利用 modelValue 作为状态接收，通过 update:modelValue 事件通知父组件。这种“声明式”的通信避免了命令式调用（如：modal.show()），让代码更符合 Vue 哲学。
      树形结构转化算法（BFS/DFS）：
  + 算法应用：手写了 arrToTree 算法。利用 Map 映射表将后端返回的扁平数组转化成层级树。理解了时间复杂度从 
    优化到 O(n) 的空间换时间策略。
+ 计算机网络：全链路错误处理
  + HTTP 状态码业务化：
    + 深度理解：在拦截器中区分了 401（未授权/Token过期）和 403（有账号但权限不足，如 Editor 访问 Admin 页面）。
  + Token 无感刷新拦截器：
    + 八股考点：异步请求的并发控制。
    + 实战方案：在 request.ts 中利用 闭包 和 Promise 队列 实现了 Token 刷新逻辑。当多个请求在 Token 过期时并发发出，通过 isRefreshing 锁将后续请求挂起，待刷新成功后一次性重新执行

+ 遇到的问题：
  + Vue 模板语法报错：在写 <BaseModal> 时，多写了冒号（@:updated:model-value）以及缺少动态绑定（title="editingItem?.id"），导致报出底层的 Unhandled error during execution of render function。
  + 解决：Vue 的模板编译非常严格，一个小小的标点失误就会引起组件树崩溃，后续编写需要更加细心审查指令符号。
  + 删除接口失效：后端 Node 服务中的删除接口，使用了 parseInt() 去转换我前端生成的含有字母的自定义 ID（如 u_abcd123），得到 NaN 导致删不掉。
  + 解决：将对比条件统一转换为字符串类型 String(u.id) === String(id)，保证了数据流通的稳健性。
