+  心得与收获(总结学习成果，要详细说明对相关知识点的个人理解.):
   +  js数字只有一种类型，0.1+0.2=0.30000000004原因是在进行进制转化时的误差
   +  typeof检测数据类型时,null，[ ] 会返回object,判断数组用Array.isArray()
   +  局部变量在声明就开始，直到执行完毕就会被销毁，全局变量是到页面被关闭再销毁，两者存在时间并不相同
   +  作用域链，顺着作用域查找遍历
   +  this指向全局指向window全局对象，根据所在环境不同指向也不同
      +  箭头函数内部没有this，只能指向父作用域
      +  对象内部的this指向构造函数,New创建的对象，会通过`obj.__proto__`指向`obj.prototype`，并且通过call(),或者appl()改变this指向到创建的实例对象上
      +  `obj.__proto__.constructor` ==构造函数,构造函数只有在new调用时才是构造函数，平时为普通函数
      +  检测实例对象的属性，先自身，再原型链，直到null
      +  own.propoty——》原型——》`原型.__proto__`——》Object.propotype——》null
         +  通过`key in obj`检测,自身加原型链
         +  obj.hasOwnProperty(key)
            +  只检测对象自身有无该属性，不检查原型链,找不到返回undefined
   +  闭包就是外部调用函数内部的参数，并且在调用结束前变量内存一直存在
   +  事件循环
      +  执行同步任务，完毕后，检测有无微任务，将微任务执行完，看有无宏任务，拿一个执行，完毕后再检测有无微任务，循环往复直到任务队列为空
      +  Js是单线程，可以节约资源时间，但是宏任务调用时间太长也会造成堵塞
   +  split,slice截取
      +  都是设置起始位置，和结束位置并且结束位置都获取不到
   +  声明和暂时性死区
      +  先调用再声明就会报错这就是暂时性死区（对应let const）;
      +  var 声明时会变量提升到全局顶部，此时访问变量会显示undefined未定义
