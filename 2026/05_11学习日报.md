## 2026/05/11 Day13 Vue3深度学习日报

+ 学习时长5 小时

+ 学习内容：
  - 完整链路巩固：从用户输入到 DOM 更新的 8 步完整链路，用 FormNode.vue 场景逐步推演：el-input emit → @update:modelValue 赋值 → Proxy set → trigger → queueJob 去重 → 微任务 effect.run → render 生成新 VNode → patch diff 双端+最长递增子序列 → DOM API 操作 → 页面更新
  - v-model 与 watch 区分纠正：v-model 展开的 @update:modelValue 回调只是简单赋值 val => formData[field] = val，不是 watch；watch 用于数据变化后做额外副作用操作
  - reactive Proxy set 陷阱纠正：不是"通知 reactive"，而是 Proxy set 主动调了 trigger
  - 生命周期完整顺序：父子嵌套首次加载 → 父 beforeMount → 子 beforeMount → 子 mounted → 指令 mounted → 子 onActivated → 父 mounted（组件先指令后，"靠近 DOM"意味着晚触发）；KeepAlive 切回只触发 onActivated，不触发 mounted
  - Mini_ref.ts 与真实 Vue 3 对比：Mini_ref 中 set value() 直接用 dep.forEach 同步执行 effect，真实 Vue 3 通过 queueJob + Set + Promise.resolve() 微任务实现去重批量调度，三次赋值只渲染一次
  - 手改 Mini_ref.ts：在 set value() 中用 queueJob 替代直接 forEach，实现 Set 去重 + 微任务批量执行

+ 心得与收获：
  今天最大的转变是从"被考到能讲"——昨天自我总结里"用户打字 → DOM 更新"第四部分还是空白的，今天能不看资料完整说出 8 步。发现之前一直混淆的一个关键点：v-model 展开的 @update:modelValue 回调不是 watch，只是一个简单的赋值函数。watch 是"数据变了之后做额外操作"（如草稿自动保存），而 v-model 回调只是把值写回去，触发响应式系统往下走。生命周期也终于理清了父子嵌套的顺序——"子先父后"是核心，指令比组件更靠近 DOM 所以晚触发（不是早触发）。Mini_ref.ts 的改造让抽象理论变成可运行的代码——在 set value() 里加 queueJob 去重排队，微任务批量执行，跟真实 Vue 3 调度逻辑完全对齐。
