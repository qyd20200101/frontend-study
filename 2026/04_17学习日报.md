## 2026/04/17 Day15 学习日报

+ 学习时长： 8小时

+ 学习内容：
  +  Vue Router 进阶应用：深入理解路由守卫（Navigation Guards）的生命周期，实现了基于“函数式初始化”的 setupRouterGuard，解决了 Pinia 实例与路由初始化顺序冲突的问题。
  + 动态权限路由（RBAC）：掌握了 router.addRoute() 的核心用法。学习了如何根据后端返回的 roles 权限列表，递归过滤异步路由表（asyncRoutes），并动态注入到 Vue 实例中。
  + Pinia 状态持久化与优化：学习使用 pinia-plugin-persistedstate 插件。重点掌握了选择性持久化（pick/paths），理解了为什么不能将动态数据（如权限、路由树）持久化的深层原因。
  + 全局进度条集成：通过 nprogress 插件提升了用户切换页面时的视觉体验。
    异常处理机制：在路由守卫中加入 try...catch 逻辑，配合 Axios 响应拦截器，实现了 Token 过期自动登出并重定向至登录页的功能。
+ 心得与收获：
  + 对“白屏”现象的深度理解：今天最大的收获是理解了单页面应用（SPA）中动态路由的“瞬时性”。动态路由是保存在内存中的，一旦 F5 刷新，路由表会重置。如果此时由于持久化插件记住了用户信息（roles），导致守卫直接放行（next），路由系统就会因为找不到对应的组件而显示白屏。解决方法是“按需持久化”——只存 Token，强制刷新后重新触发获取用户信息并重新挂载路由。
  + 路由递归过滤的逻辑：通过递归函数生成路由树，深刻体会了数据结构在前端权限控制中的重要性。不仅要考虑一级菜单，还要递归处理嵌套的 children 路由。
    导航守卫的重定向技巧：理解了 next({ ...to, replace: true }) 的精妙之处。在动态添加路由后，必须中断当前的导航并启动一次新的导航，这样新加入的路由表才能被当前路径匹配到，否则会因匹配不到而跳向 404。
+ 遇到的问题：
  + F5 刷新白屏问题：最初是因为全量持久化了 UserStore 的状态，导致守卫逻辑跳过了 addRoute 过程。通过查阅文档将 paths 优化为 pick（或版本兼容的对应属性），仅保留 Token，成功触发了刷新的自愈逻辑。
  + 404 捕获时机错误：发现刷新页面时总是跳到 404。原因是通配符路由 /:pathMatch(.*)* 被静态注册了。学习后了解到，通配符路由必须在所有动态路由 addRoute 之后再添加，否则它会先于动态路由匹配。
    持久化插件配置报错：在配置 persist 时遇到 TypeScript 报错“对象字面量只能指定已知属性”。经排查是由于插件版本升级，配置项从 paths 变更为 pick，通过查看类型定义文件（.d.ts）解决了属性名不匹配的问题。
