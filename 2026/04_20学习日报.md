## 2026/04/20 Day18学习日报
+  每天提交日报，日报格式：

+ 学习内容： 学习的知识点
    + JS执行机制
        + 单线程，调用栈，同步任务——>微任务队列——>取出一个宏任务执行——>微任务队列
        + 五万条循环卡死
            + 同步循环：霸占调用栈，在循环执行完前，浏览器无法执行其他渲染
            + 分批注入：利用`requestAnimationFrame`将任务拆分，当执行完一小段，调用栈清空是就进行`Layout`和`Paint`实现UI流畅
        + 垃圾回收机制与内存泄露
            + 防止内存泄露的方式
                + 闭包引用
                + 定时器
    + 浏览器事件流
        + 事件冒泡与捕获
            + 捕获——>目标阶段——>冒泡阶段
            + 事件委托：将事件绑定在父元素上，利用事件冒泡机制，将子元素事件传给父元素，并提供e.target识别是具体子元素，减少内存事件监听器的数量
        + stop与prevent：使用@click.stop且u但事件向上传递的路径，防止触发父元素的事件
    + HTTP缓存策略
        + 强缓存，响应头：Cache-Control:max-age=3600，让浏览器直接从本地拿资源，状态码200，主要用于项目打包的同字体文件
        + 协商缓存，响应头：ETag,Last-Modified浏览器通过ETag询问服务器资源是否改变，返回304则没改变继续使用旧的，节约带宽和解析时间
        + ETag:主要检查内容发生改变,Lat-Modified检测时间，如果一秒钟多次修改感知不到
    + 不同刷新页面过程
        + 地址栏回车：强缓存
        + F5刷新，失效强缓存，走协商缓存，跳过Cache-Control
        + Ctrl+F5，所有缓存失效，从服务器下载资源
    + Vite打包，通过index[hash].js检测哈希值是否改变，改变就下新文件
        + HTML文件设no-cache每次都协商，JS/CSS设置max-age=1年,强缓存
