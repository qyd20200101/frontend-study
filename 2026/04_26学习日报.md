## 2026/04/26 Day21学习日报

- **学习时长**：4小时
- **学习内容**：JavaScript 核心底层原理深度复盘与模拟面试（全链路联动）

### 1. 执行上下文与作用域链
- **let/const 与 var 的区别**：深入理解了在执行上下文创建阶段的差异。`var` 经历“提升 + 初始化”，而 `let`/`const` 是“提升但未初始化”，从而产生了 **TDZ（暂时性死区）**。
- **词法作用域**：明确了词法作用域是**静态**的，由函数定义的位置决定，而非调用位置。

### 2. 闭包与内存管理
- **闭包本质**：函数与其定义时的词法环境的引用关系。
- **内存释放机制**：掌握了垃圾回收（GC）的“可达性”算法。要释放闭包内存，必须断开全局对闭包函数的引用（如 `f = null`），仅操作内部变量无效。

### 3. this 绑定与 new 操作符
- **this 优先级**：`new` > 显式绑定 (`bind`/`call`) > 隐式绑定 (`obj.foo`) > 默认绑定。
- **new 的四个步骤**：
    1. 创建空对象。
    2. 链接原型（`__proto__`）。
    3. 绑定 `this`。
    4. 返回对象（若构造函数显式返回对象，则会覆盖实例）。
- **箭头函数**：没有 `[[Construct]]` 和 `prototype`，其 `this` 通过词法捕获，无法被 `new` 且对 `call`/`bind` 免疫。

### 4. 原型链与属性操作
- **链条区分**：明确区分了**变量查找**（作用域链）与**属性查找**（原型链）。
- **属性屏蔽 (Shadowing)**：在实例上写入属性不会修改原型上的同名属性。
- **delete 操作符**：只能删除对象自身的属性，无法触及原型链。

### 5. 底层优化与沙箱设计
- **V8 引擎优化**：理解了“隐藏类（Hidden Classes）”对对象属性排列顺序的要求。
- **new Function 特性**：其作用域链只包含全局对象，不产生本地闭包。结合 `.call(instance)` 可在低代码项目中实现安全的逻辑校验沙箱。

---

### 心得与收获
1. **建立了“两链一权”完整心智模型**：通过模拟面试将执行上下文、作用域链、原型链和 `this` 权限彻底串联。核心准则：“变量找作用域链，属性找原型链，this 看调用方式”。
2. **低代码系统的底层支撑**：深刻理解了 `new Function` 在动态校验中的作用，以及如何通过 `.call()` 为用户脚本注入组件实例（`this`）。
3. **理清渲染性能逻辑**：通过学习 Event Loop 微任务清空机制，明白了复杂计算必须做“时间切片”，否则微任务占据主线程会导致浏览器无法渲染，造成页面假死。

### 遇到的问题
- **问题 1：混淆“变量查找”与“属性查找”**
    - **解决**：明确不带“点”的标识符走作用域链；带“点”的访问走原型链。
- **问题 2：误以为 delete 属性会波及原型链**
    - **解决**：实验确认 delete 具有局部性，删除原型属性必须通过 `Constructor.prototype` 显式操作。
- **问题 3：对 Symbol.hasInstance 修改 instanceof 行为理解不深**
    - **解决**：通过模拟代码认识到这是 ES6 留给开发者的底层判断“后门”，在跨 iframe 判断类型或自定义类库时非常有用。
