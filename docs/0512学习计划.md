## 七、"掌握"的五级标准

> 沿袭 VUE3_STEP1 的 🟢🟡🔴 三层体系，拆细为五级，防止"以为自己会了但面试时说不出来"。

### 五个层级

| 层级 | 标志 | 定义 | 自检方式 |
|------|------|------|----------|
| **L5 能讲** | 🟢 | 不看任何资料，用白话给同事讲清楚原理，能回答追问 | 录音自己讲一遍，回听没有卡顿和模糊用词 |
| **L4 能手写** | 🟢 | 关闭所有参考，默写出核心代码（Mini 版即可，不是抄源码） | 空白文件从零写出，运行通过 |
| **L3 能看懂** | 🟡 | 看到源码/文章能理解每一步在做什么，但默写/口述会卡 | 打开 Mini_ref.ts 能逐行解释，但关了就不行 |
| **L2 有印象** | 🔴 | 知道这个概念存在，但具体内容说不清 | 脱口而出"我知道这个，但具体不太确定" |
| **L1 没听过** | 🔴 | 完全不知道，第一次接触 | — |

### 不同知识的达标线（面试导向）

并非所有知识点都要到 L5——根据面试权重，不同知识有不同的"及格线"：

| 知识类型 | 最低要求 | 原因 |
|----------|---------|------|
| 简历核心项目（低代码/虚拟滚动/MCP） | **L5 能讲** | 面试必定深挖，讲不清楚就是减分项 |
| 框架核心原理（响应式/diff/渲染链路） | **L5 能讲** | 前端面试永恒话题 |
| 框架源码级细节（scheduler/Fiber 节点结构） | **L4 能手写** | 能写 Mini 版证明理解到位 |
| 手写题（Promise/LRU/防抖节流/深拷贝） | **L4 能手写** | 面试现场可能要白板写 |
| 工程化工具（Router/Pinia/Node） | **L3 能看懂** | 能说清核心机制即可，不要求源码级 |
| 计网/安全（HTTP 缓存/CORS/XSS） | **L3 能看懂** | 能说清场景和原理，不会被问到底层协议 |
| 算法（树/链表/回溯） | **L4 能手写** | 常见面试手撕 |
| TypeScript 工具类型 | **L4 能手写** | Partial/Pick 等基本写法不能错 |

### 从 L2→L5 的升级路径

```text
🔴 L1/L2（没听过/有印象）
  │  ① 看核心文档/源码，理解"它解决什么问题"
  │  ② 用自己的话写 3 句话摘要
  ▼
🟡 L3（能看懂）
  │  ③ 手写 Mini 实现（精简到 ≤50 行，只保核心逻辑）
  │  ④ 对照真实源码，标注你的 Mini 版少了什么
  ▼
🟢 L4（能手写）
  │  ⑤ 关闭所有资料，空白文件重写 Mini 版
  │  ⑥ 在项目里找一个应用场景，验证理解
  ▼
🟢 L5（能讲）
  │  ⑦ 录音/录屏，给假想的同事讲一遍，回听卡顿点
  │  ⑧ 准备 3 个追问（"为什么这样设计？""如果去掉 X 会怎样？"），自己能答上来
```

### 每日自检格式

每天学完后在日报/自评里写：


## 八、学习原则

1. **每个知识点不求学全，求"能讲清楚"** — 面试不是笔试，能流畅口述 > 能手写但说不清
2. **Vue3 底层不再深入** — 生命周期/Teleport/Suspense 搞完就收，不挖 compiler 细节
3. **React 学习始终对照 Vue3 做对比笔记** — 每个 React 概念找到 Vue3 对应物，效率翻倍
4. **算法只补高频** — 不刷 LeetCode，只练 LRU/链表/树/Promise/虚拟滚动 五个方向
5. **每个模块学完立即自评** — 按五级标准自检，每周日统一更新
6. **碎片基础不求一次吃透** — 每天 0.5h 一个小主题，三周覆盖 15 个高频考点
7. **证据必须可证伪** — 不能写"掌握了"，必须写"能默写 XX 代码"或"能画出 XX 流程图"
