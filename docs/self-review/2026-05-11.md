## 一、我真正掌握的内容（🟢）

### 用户输入 → DOM 更新完整链路（今天最大的成果）

```
用户打字
  ↓
① el-input emit(''update:modelValue'', 新值)
  ↓
② @update:modelValue 回调 → formData[comp.field] = 新值
  ↓
③ Proxy set 陷阱 → trigger(formData, comp.field)
  ↓
④ trigger 找到所有依赖此字段的 effect → queueJob(effect) → Set 去重
  ↓
⑤ 同步代码结束 → 微任务执行 → effect.run() → render() → 新 VNode 树
  ↓
⑥ patch(旧VNode, 新VNode) → 双端对比 + 最长递增子序列
  ↓
⑦ 根据 diff 结果调用 DOM API
    - 值变了 → el.value = 新值 / el.textContent = 新文本
    - 移动位置 → parent.insertBefore(el, refEl)
    - 删除 → parent.removeChild(el)
    - 新增 → parent.appendChild(newEl)
  ↓
⑧ 页面更新完成。nextTick(() => { ... }) 在此执行。
```

### 生命周期钩子顺序（今天补完）

**父子组件首次加载：**
父 beforeMount → 子 beforeMount → 子 mounted → 子 onActivated → 父 mounted

**切走再切回（KeepAlive 缓存）：**
只用 onActivated — 没有 beforeMount、没有 mounted，组件实例还活着

**指令插入位置：**
组件 mounted → 指令 mounted → onActivated → 父 mounted
"靠近 DOM"意味着晚触发，不是先触发

## 二、今天纠正的核心错误

| 之前的误解 | 纠正后的正确理解 |
|---|---|
| v-model 回调是 watch | v-model 展开的 @update:modelValue 回调只是一个简单赋值 val => formData[field] = val，不是 watch |
| watch 依赖变就执行 | watch 用于"数据变了之后做额外操作"（副作用），而 v-model 回调只是传值 |
| reactive 被通知 | 不是被通知 — reactive 对象的 Proxy set 陷阱主动调了 trigger |
| 指令 mounted 先于组件 | 实际上是组件 mounted 先，指令 mounted 后（指令需要在元素已插入 DOM 后才执行） |
